# agent-rec

Local-first flight recorder for coding agents.

`agent-rec` captures what terminal coding agents actually do: shell output, exit status, git diffs, changed files, Claude Code hook events, and run metadata. It writes portable JSONL traces, static HTML reports, and scrubbed repro bundles that can be attached to bug reports, PRs, or future replay tools.

## Why this exists

Generic LLM observability shows spans, model calls, cost, and latency. Coding-agent debugging needs repo-native evidence:

- What command ran?
- What changed in git?
- Did tests run?
- What did the terminal output show?
- What state did the workspace end in?
- Which agent tool/hook events occurred?
- Can I safely share the run with secrets scrubbed?

`agent-rec` is meant to become a small, open, local artifact format for coding-agent runs: something maintainers can inspect, attach to issues, review in PRs, and eventually replay.

## Install for development

```bash
git clone https://github.com/isanchez404/agent-rec.git
cd agent-rec
uv sync --extra dev
uv run agent-rec --help
```

## Quick start

Record any command:

```bash
uv run agent-rec run -- python -c "print('hello from agent-rec')"
```

Inspect the latest trace:

```bash
latest=$(ls -t .agent-rec/sessions/*.jsonl | head -n 1)
uv run agent-rec inspect "$latest"
```

Generate a static HTML report:

```bash
uv run agent-rec report "$latest" --out report.html
open report.html
```

Create a scrubbed repro bundle:

```bash
uv run agent-rec bundle "$latest" --out repro.tar.gz
```

The bundle contains:

- `trace.jsonl`: scrubbed portable event trace
- `report.html`: scrubbed static report
- `README.md`: bundle metadata

## CLI

```bash
agent-rec run -- <command> [args...]
agent-rec inspect .agent-rec/sessions/<session>.jsonl
agent-rec report .agent-rec/sessions/<session>.jsonl --out report.html
agent-rec bundle .agent-rec/sessions/<session>.jsonl --out repro.tar.gz
agent-rec claude-hook --trace <trace.jsonl> --session-id <agent-rec-session-id>
```

## Trace events

The initial schema is JSONL with one event per line:

```json
{
  "schema": "agent-rec.trace.v0",
  "type": "process.output",
  "session_id": "20260512T161310Z-22df2411",
  "timestamp": "2026-05-12T16:13:10Z",
  "data": {"stream": "stdout", "text": "hello\n"}
}
```

Current event types:

- `session.started`
- `process.started`
- `process.output`
- `process.exited`
- `git.snapshot`
- `session.finished`
- `claude_code.hook`

## Claude Code hook ingestion

Claude Code hooks can append richer tool-use metadata to an existing trace. The hook command reads Claude's JSON payload from stdin:

```bash
agent-rec claude-hook \
  --trace .agent-rec/sessions/<session>.jsonl \
  --session-id <agent-rec-session-id>
```

The normalized event type is `claude_code.hook` and includes:

- Claude session id
- hook event name
- tool name
- scrubbed tool input
- cwd
- transcript path
- permission mode

## Secret scrubbing

Reports and bundles scrub common sensitive values by default, including:

- OpenAI-style `sk-...` keys
- Anthropic `sk-ant-...` keys
- GitHub `ghp_...` / related tokens
- bearer tokens
- AWS access-key ids
- values under key names like `api_key`, `token`, `secret`, `password`, or `authorization`

The raw trace generated by `agent-rec run` remains local evidence. `agent-rec bundle` writes a scrubbed `trace.jsonl` and scrubbed `report.html` into the shareable archive.

5. Trace schema stabilization and exporters.
