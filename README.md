# agent-rec

Local-first flight recorder for coding agents.

`agent-rec` captures what terminal coding agents actually do: prompts, shell output, exit status, git diffs, changed files, Claude Code hook events, and run metadata. It writes portable JSONL traces, static HTML reports, and scrubbed repro bundles that can be attached to bug reports, PRs, or future replay tools.

## Why

Generic LLM observability shows spans and model calls. Coding-agent debugging needs repo-native evidence:

- what command ran
- what changed in git
- whether tests ran
- what the terminal output showed
- what state the workspace ended in
- which tool/hook events occurred
- whether shareable artifacts were scrubbed before distribution

## MVP commands

```bash
agent-rec run -- <command> [args...]
agent-rec inspect .agent-rec/sessions/<session>.jsonl
agent-rec report .agent-rec/sessions/<session>.jsonl --out report.html
agent-rec bundle .agent-rec/sessions/<session>.jsonl --out repro.tar.gz
```

## Claude Code hook ingestion

Claude Code hooks can append richer tool-use metadata to an existing trace. The hook command reads Claude's JSON payload from stdin:

```bash
agent-rec claude-hook \
  --trace .agent-rec/sessions/<session>.jsonl \
  --session-id <agent-rec-session-id>
```

The normalized event type is `claude_code.hook` and includes:

- Claude session id
- hook event name
- tool name
- scrubbed tool input
- cwd
- transcript path
- permission mode

## Secret scrubbing

Reports and bundles scrub common sensitive values by default, including:

- OpenAI-style `sk-...` keys
- Anthropic `sk-ant-...` keys
- GitHub `ghp_...` / related tokens
- bearer tokens
- AWS access-key ids
- values under key names like `api_key`, `token`, `secret`, `password`, or `authorization`

The raw trace generated by `agent-rec run` remains local evidence. `agent-rec bundle` writes a scrubbed `trace.jsonl` and scrubbed `report.html` into the shareable archive.

## Status

Early OSS scaffold. The first wedge supports generic CLI recording using subprocess capture plus git snapshots, static diff/output reports, scrubbed repro bundles, and Claude Code hook ingestion. Future adapters can add richer OpenCode/Hermes hooks and replay workflows.
